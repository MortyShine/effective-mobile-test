## Задание 3: Архитектура PUSH-уведомлений

### 1. Компоненты архитектуры
Для реализации гибкой отправки пушей (по триггерам, брошенным корзинам и маркетинговым рассылкам) необходимы следующие микросервисы:
* **Брокер сообщений (Kafka/RabbitMQ):** Шина для обмена событиями между сервисами.
* **Сервис уведомлений (Notification Service):** Бизнес-логика пушей. Принимает событие, определяет шаблон текста и отправляет запрос провайдерам.
* **База данных пушей (Notification DB):** Хранит историю отправлений, статусы (доставлено/прочитано) и шаблоны сообщений.
* **База токенов (Device Token Storage):** Хранит связки `user_id <-> device_token` для отправки пуша на конкретное устройство.
* **Внешние провайдеры (Push Gateways):** Сервисы доставки сообщений на ОС (APNs для iOS, FCM для Android).

---

### 2. Сценарии и логика работы схемы

#### Сценарий А: Триггерный пуш (Пример: «Отмена заказа»)
1. **Сервис Заказов** меняет статус заказа на «Отменен» и отправляет событие `order_cancelled` в брокер сообщений.
2. **Сервис уведомлений** подписан на эту тему. Он мгновенно считывает событие из брокера.
3. Сервис запрашивает актуальный `device_token` пользователя из базы токенов.
4. Сервис берет нужный шаблон текста, собирает пуш и отправляет его во внешние шлюзы (APNs/FCM).
5. Провайдеры (Apple/Google) доставляют пуш в мобильное приложение «Петрушка Зеленая».

#### Сценарий Б: Отложенный пуш (Пример: «Забытая корзина»)
1. **Крон-планировщик (Scheduler)** или специализированный воркер раз в час сканирует базу корзин.
2. Если корзина не менялась более X часов, он генерирует событие `cart_abandoned` в брокер.
3. Дальнейшая цепочка аналогична Сценарию А: Сервис уведомлений подхватывает событие -> находит токен -> шлет через APNs/FCM.

---

### 3. Описание текстовой блок-схемы (Архитектурные связи)

```text
[ Бизнес-сервисы: Заказы / Корзина / Маркетинг ]
                       │
                       │ (Публикация событий: order_cancelled, cart_abandoned)
                       ▼
      [ Брокер сообщений (Kafka / RabbitMQ) ]
                       │
                       │ (Асинхронное чтение событий)
                       ▼
   [ Микросервис уведомлений (Notification Service) ]
       ├──► [ База данных токенов (Device Tokens) ]
       └──► [ База шаблонов и истории пушей ]
                       │
                       │ (Запрос на отправку с токеном устройства)
                       ▼
   [ Внешние шлюзы отправки: Google FCM / Apple APNs ]
                       │
                       │ (Доставка Push на устройство)
                       ▼
         [ Мобильное приложение пользователя ]
```
